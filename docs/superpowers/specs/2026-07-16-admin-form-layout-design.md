# 登录与后台新建表单布局设计

## 目标

修复登录页字段出现双层边框的视觉问题，并统一后台“新建账号”“新建商品”两个居中弹窗的表单层次、间距和操作区。

## 范围

- 登录页只保留字段容器的一层边框、背景和焦点态；内部 `input` 继承透明背景且不再附加通用卡片输入样式。
- “新建账号”弹窗采用较宽的单列表单：账号信息、权限设置、商品分配三个区块；商品列表可滚动。
- “新建商品”弹窗采用两列网格：ID、名称、价格、币种、链接为短字段，介绍独占整行。
- 两个弹窗使用同一套标题、关闭按钮、字段标签、错误提示与底部操作栏；小屏幕下回退为单列，并保证内容可滚动。

## 不变项

- 登录、创建账号、创建商品的接口参数、校验和提交行为不变。
- 编辑商品、重置密码和续期弹窗不改变交互语义，仅可复用通用的弹窗样式。

## 组件与样式边界

- `StaticsPage.css` 管理登录页与后台通用的弹窗基础样式。
- `UsersTab.tsx` 与 `ProductsTab.tsx` 只增加表达表单分区和操作栏的语义类名。
- `UsersTab.css` 管理账号商品分配列表的局部布局；不再把账号表格样式放入全局样式。

## 交互与可访问性

- 点击遮罩或关闭按钮均关闭未提交的弹窗，保持现有行为。
- 取消按钮与关闭按钮行为一致；提交按钮在原有无效或提交中状态保持禁用。
- 表单控件保留可见焦点态和既有 `label` 关联。

## 验证

- 现有组件测试通过；为表单结构增加可定位的回归断言。
- TypeScript 类型检查与生产构建通过。
