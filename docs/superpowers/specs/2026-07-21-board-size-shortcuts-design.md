# 拼豆板尺寸快捷选择与工时预估 Design

> **For:** 拼豆图生成器（pingdou）
> **Date:** 2026-07-21
> **Status:** Approved for planning

## 目标

让用户能一键选择常见拼豆板尺寸，并基于当前图案的实际主体豆子数看到制作时间预估。导出图按所选板子尺寸生成正方形、透明补边的网格图。

支持的快捷尺寸：`52 × 52`、`78 × 78`、`104 × 104`。

## 交互与展示

### 桌面端

在 `ParamPanel` 的“网格大小（长边豆子数）”滑杆上方增加“快捷板子尺寸”按钮组：

- 每个按钮显示 `52 × 52`、`78 × 78` 或 `104 × 104`。
- 点击按钮调用已有的 `onGridSizeChange(size)`，沿用现有重算流程。
- 当前网格值等于按钮尺寸时显示选中状态。
- 保留滑杆和既有预设，用户仍可使用滑杆精细选择其他网格大小。
- 修正采样尺寸计算：`gridSize` 代表图案最长边的格数。选择 `52 × 52` 时，无论原图横竖，输出图案最长边均不超过 52 格，可放入对应规格的正方形板子。
- 导出时，所选尺寸同时决定导出的网格画布边长。选择 `52 × 52` 时导出图为 52 × 52 格；非正方形图案在正方形画布内居中，空余格透明。

### 移动端

在 `MobileActionBar` 网格控制条下方增加同一组紧凑按钮：

- 点击时调用已有 `onGridSizeChange(size)`。
- 高亮规则与桌面端一致。
- 不改变固定底部操作栏的上传、去背景和导出能力。

### 制作时间

在当前豆子统计区域显示预计时间：

```
约 X.X 小时
```

- 仅当 `beanCount > 0` 时显示。
- 公式：`beanCount / 600`，结果保留一位小数。
- `beanCount` 已排除自动去背景掩码格；切换去背景、调整网格或选择板子后，工时自动更新。
- 600 颗/小时为统一前端常量，不引入服务端数据或配置项。

## 数据流

```text
快捷板尺寸按钮
  → onGridSizeChange(size)
  → App.reprocess({ gridSize: size, ... })
  → PipelineResult / beanCount 更新
  → estimateHours(beanCount)
  → 桌面端 / 移动端显示“约 X.X 小时”

导出按钮
  → exportMulti(selectedGridSize)
  → 以所选尺寸重采样图案
  → 居中写入 selectedGridSize × selectedGridSize 透明画布
  → 下载 pingdou-{selectedGridSize}x{selectedGridSize}.png
```

## 代码结构

- 新增一个纯函数模块（例如 `src/pipeline/timeEstimate.ts`），负责将豆子数转换为小时数，避免在两个 UI 组件中重复计算。
- `ParamPanel` 和 `MobileActionBar` 复用同一个板子尺寸常量，避免双份列表漂移。
- `App` 继续持有 `gridSize` 与 `beanCount`；导出调用显式传入已选尺寸，避免将矩形处理结果的短边误作导出尺寸。
- `Pipeline` 负责生成透明正方形导出画布；预览与制作豆子统计继续基于实际图案矩形网格。

## 边界与错误处理

- `beanCount <= 0` 时返回 `null`，不展示工时。
- 所有快捷尺寸都是既有 `gridSize` 的合法正整数。
- 非正数、非有限豆子数由纯函数保守处理为 `null`。
- 透明补边仅用于导出画布，不写入豆子统计或配色表。

## 测试

- 工时纯函数：0、负数、非有限值、600 颗、900 颗的显示值。
- 桌面端：三个快捷按钮可触发回调、与当前尺寸同步高亮、显示预计小时。
- 移动端：三个快捷按钮可触发回调、与当前尺寸同步高亮、显示预计小时。
- 导出：横竖图均使用所选尺寸生成正方形图、主体居中、补边透明，文件名包含所选正方形尺寸。
- 现有网格滑杆、去背景与导出测试保持通过。

## 非目标

- 不根据用户熟练度、图案颜色数或板子数量动态改变“600 颗/小时”的速度。
- 不新增自定义板子尺寸输入框。
- 不改变预览的图像采样、量化或背景去除算法。
